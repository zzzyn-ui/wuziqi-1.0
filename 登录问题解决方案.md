# 五子棋登录问题 - 完整分析报告

## 问题描述
前端点击登录按钮后一直在转圈，没有响应。注册功能也是同样的问题。

## 根本原因分析

### 1. 端口配置 Bug（主要问题）
**文件**: `src/main/java/com/gobang/core/netty/NettyServer.java`

**问题代码**（第 87 行）:
```java
public NettyServer(String host, int port, ...) {
    port = 8083;  // ❌ 强制覆盖传入的端口！
    this.host = host;
    this.port = port;  // 使用的是被覆盖后的 8083
    ...
}
```

**影响**:
- 配置文件 `application.yml` 设置端口为 9090
- 但代码强制将其改为 8083
- 前端连接 9090，服务器监听 8083，导致连接失败

**修复状态**: ✅ 已修复

### 2. 消息路由问题
**问题**: `type=1` (AUTH_LOGIN) 消息没有被正确路由到 `AuthHandler`

**原因分析**:
1. WebSocket 连接成功（101 状态码）
2. 消息发送成功
3. 但消息处理流程中存在问题

**消息处理流程**:
```
WebSocketHandler.channelRead()
  ↓
handleJsonFrame() - 解析 JSON
  ↓
convertJsonBodyToProtobuf() - 转换 body
  ↓
handlers.get(AUTH_LOGIN) - 获取 handler
  ↓
AuthHandler.handle() - 处理登录
  ↓
handleLogin()
  ↓
authService.login() - 数据库验证
  ↓
ResponseUtil.sendResponse() - 发送响应
```

### 3. 调试日志缺失
原代码缺少关键调试日志，无法追踪消息处理流程。

## 常见问题

### Q: 端口仍被占用怎么办？
```cmd
# 查找占用端口的进程
netstat -ano | findstr ":9090"

# 强制结束进程
taskkill /F /PID <进程ID>
```

### Q: Redis 连接错误怎么办？
Redis 错误不会影响登录功能，登录使用的是数据库。如需修复 Redis：
```cmd
# 检查 Redis 是否运行
redis-cli ping

# 启动 Redis
redis-server
```

### Q: 数据库连接错误怎么办？
1. 检查 MySQL 是否运行
2. 检查配置文件 `src/main/resources/application.yml` 中的数据库配置
3. 确保数据库 `gobang` 存在
